# Contribution Guidelines

Welcome to the DEEPFAKE-GUARD project! We appreciate your interest in contributing to our repository. Please follow these guidelines to ensure a smooth contribution process.

## Code of Conduct

We expect all contributors to uphold our code of conduct by treating everyone with respect and kindness. Discrimination, harassment, or any negative behavior will not be tolerated.

## Development Setup

1. **Fork the Repository**: Click the 'Fork' button on the top right of the repository page.
2. **Clone Your Fork**: Use the command `git clone https://github.com/your-username/DEEPFAKE-GUARD.git` to clone your fork locally.
3. **Install Dependencies**: Navigate to the project directory and run `npm install` to install all required dependencies.

## Contribution Types

You can contribute in various ways:
- **Bug Reports**: Report any bugs or issues you encounter.
- **Feature Requests**: Suggest new features to enhance the functionality of the project.
- **Code Contributions**: Submit pull requests with your code improvements.

## Pull Request Process

1. **Create a New Branch**: Create a branch from `main` for your changes: 
   ```
   git checkout -b feature/your-feature-name
   ```
2. **Make Your Changes**: Implement your feature or fix the bug.
3. **Commit Your Changes**: Write a clear commit message and commit your changes:
   ```
   git commit -m "Add new feature"
   ```
4. **Push to Your Fork**: Push your changes to your fork:
   ```
   git push origin feature/your-feature-name
   ```
5. **Create a Pull Request**: Go to the original repository and create a pull request.

## Coding Standards

Please ensure that your code follows the project's coding standards. This includes: 
- Use camelCase for variables and functions.
- Indent code with 4 spaces.
- Write comments for complex logic.

## Commit Message Guidelines

- Use the format: `type(scope): description`. 
  - **Type**: feat, fix, docs, style, refactor, perf, test, chore.
  - **Scope**: the area of the code affected.
  - **Description**: a short description of your changes.

## Testing

Ensure that your changes are tested. Write unit tests for new features and ensure all existing tests pass. To run tests, use:
```bash
npm test
```

## Documentation Standards

Update the documentation where necessary. Use clear and concise language. Make sure to add examples where applicable.

## Help Resources

If you need help, please reach out on GitHub Discussions or contact the maintainers.

Thank you for contributing!
